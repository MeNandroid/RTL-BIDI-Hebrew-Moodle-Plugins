Metadados DCMES

Metadata is “data about data”. Metadata is structured data set that describes, explains and facilitates the localization and recovery of information.
The Dublin Core metadata (DCMES - Dublin Core Metadata Element Set) standard is a simple yet effective element set for describing a wide range of networked resources. Some of the resources and activities in a moodle course can be annotated with DCMES, because some of the LOs ( Learning Objects) are articles, papers, posters, tutorials and other similar documents or bibliographic references. It is desirable that the metadata creation occurs when the LO (resource/activity) creation and that the LO’s author is also the author of the LO metadata.

Simple Dublin Core comprises fifteen elements (called Core Elements):

  • Title (The name given to the resource.);
  • Creator (An entity primarily responsible for making the content of the resource.);
  • Subject (The topic of the content of the resource.);
  • Description (An account of the content of the resource. Example: abstract or table of contents,);
  • Publisher (The entity responsible for making the resource available.);
  • Contributor (An entity responsible for making contributions to the content of the resource.);
  • Date (A date associated with an event in the life cycle of the resource.);
  • Type (The nature or genre of the content of the resource);
  • Format (The physical or digital manifestation of the resource: media-type or dimensions of the resource);
  • Identifier (An unambiguous reference to the resource within a given context: URI, ISBN, DOI.);
  • Source (A Reference to a resource from which the present resource is derived.);
  • Language ( A language of the intellectual content of the resource: pt, en, fr);
  • Relation (A reference to a related resource.);
  • Coverage (The extent or scope of the content of the resource.);
  • Rights (Information about rights held in and over the resource.).

Simple Dublin Core only includes these fifteen elements.

Qualified Dublin Core includes three additional elements (Audience, Provenance and RightsHolder), as well as a group of element refinements (also called qualifiers) that refine the semantics of the elements in ways that may be useful in resource discovery

Both, Simple DC and Qualified DC, can be expressed in XML and RDF.

For more information see: http://www.dublincore.org
